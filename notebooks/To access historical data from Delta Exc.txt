Historical Data Endpoints
1. Historical Candles (OHLC Data)

Endpoint: GET /v2/history/candles

This is the primary endpoint for historical price data, providing OHLC (Open-High-Low-Close) candles.

Key Features:

    Resolutions available: 1m, 3m, 5m, 15m, 30m, 1h, 2h, 4h, 6h, 1d, 1w
    Maximum candles per request: 2000
    Data types: Traded price, Mark price, Funding rate, Open Interest (OI)

Parameters:

    symbol (required): Product symbol (e.g., BTCUSD)
        For funding history: FUNDING:BTCUSD
        For mark price: MARK:BTCUSD
        For OI data: OI:ETHUSD
    resolution (required): Time frame (e.g., "5m", "1h", "1d")
    start (required): Start time (Unix timestamp in seconds)
    end (required): End time (Unix timestamp in seconds)

Example:
curl -X GET "https://api.india.delta.exchange/v2/history/candles?symbol=BTCUSD&resolution=1h&start=1685618835&end=1722511635" \
  -H "Accept: application/json"

2. Real-Time Data via WebSocket

For continuous data collection, you can use WebSocket channels:

Candlestick Channel: candlestick_{resolution}

    Provides real-time OHLC updates
    Resolutions: 1m, 3m, 5m, 15m, 30m, 1h, 2h, 4h, 6h, 12h, 1d, 1w, 2w, 30d

All Trades Channel: all_trades

    Provides tick-by-tick trade data
    Includes price, size, buyer/seller roles, timestamps
